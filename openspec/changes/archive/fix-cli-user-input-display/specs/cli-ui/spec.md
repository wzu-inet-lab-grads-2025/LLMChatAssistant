# CLI 用户界面规范

## 已有需求

### 需求：用户输入提示
系统应显示 "User>" 提示符，等待用户输入消息。

#### 场景：显示输入提示
- **当** 系统等待用户输入时
- **那么** 显示 "User>" 提示符

#### 场景：获取用户输入
- **当** 用户在提示符后输入文本并按回车
- **那么** 系统获取用户输入的文本内容

### 需求：用户消息显示
系统应使用格式化的 Panel 框显示用户消息。

#### 场景：显示用户消息
- **当** 用户输入消息后
- **那么** 显示包含用户消息的格式化 Panel（╭─ USER ─╮ 样式）

## 修改需求

### 需求：用户输入显示优化
系统应在用户输入后清除输入行，只显示格式化的 Panel 框。

#### 场景：输入后清除提示符
- **当** 用户在 "User>" 提示符后输入文本并按回车
- **那么** 系统清除包含 "User>" 的输入行
- **并且** 直接显示格式化的 Panel 框（╭─ USER ─╮）

#### 场景：避免重复显示
- **当** 用户输入消息后
- **那么** 不应同时显示原始的 "User> 输入内容" 和格式化的 Panel
- **并且** 只显示一个格式化的 Panel 框
