### 版本控制与测试纪律

- [ ] 每个阶段完成后通过真实测试

- [ ] 提交消息清晰描述阶段工作和测试结果
- [ ] 不允许跳过测试或使用虚假测试

### 安全第一原则
- [ ] 文件访问使用统一的路径白名单控制
- [ ] 白名单配置存储在 config.yaml 的 file_access.allowed_paths
- [ ] 路径验证器防止路径遍历攻击（../ 规范化）
- [ ] 黑名单模式保护敏感文件（.env、.ssh/*、/etc/passwd 等）
- [ ] 任何工具不绕过白名单访问文件系统

### 自动化与按需索引
- [ ] RAG 工具支持按需索引（execute_async 方法）
- [ ] 索引管理器使用懒加载策略，避免重复索引
- [ ] 索引持久化存储到 storage/vectors/ 目录
- [ ] 索引创建验证文件类型（仅文本文件）和大小（最大 10MB）
- [ ] 系统重启后自动加载已持久化的索引

### 多层防御策略
- [ ] 文件访问依次通过白名单、黑名单、大小限制、内容类型验证
- [ ] 命令执行限制输出大小（默认最大 100KB）
- [ ] 命令参数验证黑名单字符（;, &, |, >, <, `, $, (, ), \n, \r）
- [ ] 正则表达式搜索验证复杂度，防止 ReDoS 攻击
- [ ] Glob 模式匹配限制最大文件数（默认 100 个）

### 可审计性与透明性
- [ ] 所有文件访问操作记录日志
- [ ] 所有索引创建记录详细信息（文件路径、分块数量、创建时间）
- [ ] 所有工具调用记录执行状态、持续时间
- [ ] 文件访问被拒绝时返回明确的拒绝理由
